Архитектура проекта

Основной скелет проекта сделан на FSM. 

Последовательность работы

    Bootstrap сцена
        Через Bootstrapper поднимается и обслуживается стейт машина

    InitializingState
        Инициализируются UIManager
        Переходит в меню

    MenuGameplayState
        Ожидает кнопку старта

    LevelLoadState
        Загружается сцена уровня.
        
    GameplayState
        Через конфиг уровня создает спавнер,
        В апдейте обновляет животных и спавнер
        Поднимает свой ЮИ, в котором введет счётчик и всплывающие надписи

Логика животных:

    Логика животных разделена на три блока:
      взаимодействие с другими животными
      выбор направления
      тип перемещения
    Каждый из типов задается конфигом в AnimalDTO. То есть реализовав перемещение прыжками, можно создать прыгающую змею, или ходящую лягушку. Сейчас сделано так, что AnimalDTO хранятся в конфиге животного AnimalSO
    Конфиг уровня содержит сцену и набор возможных животных.
